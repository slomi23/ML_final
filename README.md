# ML_final
wandb link: https://wandb.ai/slomi23-free-university-of-tbilisi-/projects
(თითოეულ არქიტექტურას თავისი პროექტი აქვს, თუ არ ვცდები N-beats ის გარდა, რომლის რანებიც default პროექტში შეგიძLიათ ნახოთ)

მოცემული პროქტის მიზანი იყო Walmart-ის სხვადასხვა რეგიონში მდებარე 45 მაღაზიის გაყიდვების პროგნოზირება. პროექტზე მუშაობისას დავტესტეთ მეთოდები: 
XGBoost, LightGBM, Prophet, TFT, N_beats, DLinear
გადმოგვეცემოდა ფაილები : train.csv მთავარი სატრენინგო ფაილი 5 სვეტით : Store, Dept, Date, Weekly_Sales (target), isHoliday. test.csv სატესტო ფაილი. იგივე რაც train.csv Weekly_Sales გარეშე. features.csv - დამატებითი მონაცემებისთვის, როგორიცაა ტემპერატურა, საწვავის ფასი, ფასდაკლების კვირეული, უმუშევრობისა და სამომხმარებლო ფასების ინდექსი. ასევე გადმოცემული გვქონდა 4 კონკრეტული დღესასწაულის თარიღები: Super Bowl, Labor Day, მადლიერების დღე და შობა.

# მონაცემთა ინჟინერიის საკითხები
არქიტექტურებს შორის ყველაზე ინტენსიური feature engineering საჭირო იყო XGBoost-ისა და LightGBM-ისთვის, რადგან ეს მოდელები არ ხვდებიან დროში სერიების სტრუქტურას ავტომატურად და საჭიროებენ ხელით შექმნილ ლაგებს (lags) და სეზონურობას.

ჩვენ შევქმენით შემდეგი ფუნქციონალური მახასიათებლები:

Lag Features: დავამატეთ წინა პერიოდების გაყიდვები, როგორიცაა sales_lag_1 (წინა კვირა), sales_lag_4 (4 კვირის წინ), sales_lag_52 (წინა წლის იგივე კვირა წლიური სეზონურობის დასაჭერად).
Rolling Statistics: გამოვთვალეთ მოძრავი საშუალო (roll_mean_4, roll_mean_8) და სტანდარტული გადახრა (roll_std_4) ბოლო 4 და 8 კვირისთვის, რათა მოდელს ჰქონოდა წარმოდგენა მოკლეমেდიან ტრენდებზე.
Calendar Features: Date სვეტიდან ამოვიღეთ Year, Month, DayOfWeek და WeekOfYear.

# Deep Learning 
მოდელებისთვის (N-BEATS, TFT, DLinear): ამ არქიტექტურებისთვის მონაცემები მოვამზადეთ pytorch-forecasting ბიბლიოთეკის TimeSeriesDataSet-ის გამოყენებით. აქ feature engineering ნაკლებად ხდება ხელით. TFT-ს მაგალითად, შეუძლია გამოიყენოს სტატიკური (მაღაზიის ტიპი) და დროში ცვლადი ცნობილი ცვლადები (ტემპერატურა, დღესასწაული).

# ექსპერიმენტები და შედეგები
ექსპერიმენტები დავლოგეთ wandb პლატფორმაზე. თითოეული არქიტექტურისთვის შევქმენით ცალკე პროექტი,რანებში შენახულია ჰიპერპარამეტრები, loss მრუდები და საბოლოო მეტრიკები.

რთული საკითხი იყო ის, თუ რა უნდა გვექნა იმ ველებზე, რომელიც შესავსები გვქონდა სატესტ მასალაში (lag & rolling features). XGBoostსა და LightGBM ის შემთხვევაში გადავწყიტეთ გაგვეკეთებინა რეკურსიული ფრედიქშენი, ანუ დაგვეგენირირებია პასუხი კვირა 1თვის და მერე ეს პასუხი განმოგვეყენებიან კვირა 2თვის და ა.შ. 
ამ ყველაფრის გამო რთული იყო მოდელების შეფასება მხოლოდ ვალიდაციის შედეგით, ვინაიდან ვალიდაციის სეტზე ვიცოდით lag & rolling features.  საუკეთესო შედეგები XGBoost მა აჩვენა და WMAE score kaggle ზე იყო 4963. საბმიშენის დაგენერირება model_inference ში ხდება 

N-BEATS / TFT / DLinear: ეს მოდელები იყენებენ "one-shot" პროგნოზირებას (ყველა კვირას ერთდროულად წინასწარმეტყველებენ), რაც ამცირებს შეცდომის დაგროვების რისკს, მაგრამ საჭიროებს მეტ გამოთვლით რესურსს და კარგ ნორმალიზაციას.
